# 마크다운 문법

파일 확장자가 .md로 된 파일에 쓰이며
README.md가 대표적인 마크다운 파일입니다.

일반 텍스트 문서에 서식요소(스타일 요소)를 추가할 수 있는 가벼운 \*\*마크업 언어입니다.

\*\*마크업 언어 종류 : HTML, XML, LATeX

존 그루버가 2004년에 개발한 마크다운은 전 세계에서 가장 널리 사용되는 마크업 언어 중 하나입니다.

해당 마크업 파일을 보려면 window 기준 ctal + shift + v 를 누르거나, 해당 파일에서 우측 클릭 open preview를 하면 볼 수 있습니다.

**장점**

- 간결하다
- 별도의 도구없이 작성가능
- 다양한 형태로 변환 가능
- 텍스트로 저장되기 때문에 용량이 적다
- 텍스트 파일이기 때문에 버전관리 시스템을 이용하여 변경이력 관리 용이
- 지원하는 프로그램과 플랫폼이 다양

**단점**

- 표준이 없어 사용자마다 작성 문법이 조금씩 다를 수 있음
- 모든 HTML 마크업을 대체하지 못함

우리는 다양한 곳에서 사용할 수 있습니다
지금 사용하는 vscode부터 슬랙, 노션등에서도 사용이 가능합니다.

그리고 앞으로 배울 HTML과 CSS를 적용할 수 있기 때문에 스타일에 현혹되지 말고 각 문법의 의미에 맞게 써야한다고 할 수 있다.( 시멘틱 마크업 )

## 마크다운 문법

### 헤더

헤더는 총 6가지가 있습니다.

```
<h1><h2><h3><h4><h5><h6>
```

> # header1

- #으로 시작한다.
- #은 하나에서 여섯개까지 가능
  - h1 ~ h6 header1 ~ 6단계
- #가 늘어날때마다 제목의 크기가 줄어듦
- H1은 ===으로도 만들 수 있음
- H2는 ---으로도 만들 수 있음

### 텍스트 스타일링

#### 볼드

굵은 글자로 포인트를 주고 싶을때 사용합니다.

```
**이것은 굵은 글씨입니다**
__이것은 굵은 글씨입니다__
```

> _굵은 글자_

#### 이탤릭체

기울인 글자로 포인트를 주고 싶을때 사용합니다

```
*이것은 기울인 글자입니다*
_이것은 기울인 글자입니다_
```

> _기울인 글자_

#### 취소선

잘못된 텍스트를 표시하고 싶을때 사용합니다

```
~~ 잘못된 텍스트 입니다 ~~
~ 잘못된 텍스트 입니다 ~
```

> ~~잘못된 텍스트 입니다~~

#### 볼드 및 이텔릭폰트

볼드 및 이텔릭 폰트입니다. 매우 강조할때 사용합니다.

```
**이글은 _중요한_ 글입니다**
```

> **매우 _중요한_ 글입니다**

#### 전부 볼드, 이텔릭체

전부 중요한 글씨일때 사용합니다.

```
*** 전부 중요한 글입니다.
```

> **_전부 중요한 글입니다._**

#### 아래첨자

아래 첨자입니다.

```
이글은 <sub>아래첨자</sub> 글입니다.
```

> 이글은 <sub>아래첨자</sub> 글입니다.

#### 밑줄

밑줄을 그어 강조하고 싶을때 사용합니다.

```
이글은 <ins>밑줄</ins>입니다
```

> 이글은 <ins>밑줄</ins>입니다

### 텍스트 인용

.을 사용하여 텍스트를 인용할 수 있습니다.

```
> 인용
>> 인용의 인용
>>> 인용의 인용의 인용
```

> 텍스트 인용
>
> > 인용의 인용
> >
> > > 인용의 인용의 인용

- 텍스트를 인용하면 왼쪽에 세로줄로 들여쓰기가 됨
- 회색 글꼴로 표시

### 코드 인용

코드나 명령을 백틱(\n) 하나로 표현 가능합니다. 백틱 안의 텍스트는 서식이 적용되지 않습니다.

> 안녕하세요 `예시` 코드입니다

코드나 텍스트를 별도의 블록으로 뺄려면 백틱을 세개(```) 사용합니다.

````
    ```
        git status
        git add
        git commit
    ```
````

또한 `\`기호와 함께 사용하면 백틱 기호를 출력 할 수 있습니다.

### 특수문자

특수문자를 표현할 때, 표시될 문자 앞에 \를 넣고 특수문자를 쓰면 됩니다.

```
\* 안녕하세요
\- 안녕하세요
```

\- 안녕하세요 \* 안녕하세요

### 목록

`<ol><li><ul>`로 변환되는 목록입니다. 들여쓰기를 통해 하위 목록을 작성할 수 있습니다.
혼용 가능합니다.

```
    1. 순서
    1. 순서
        1. 순서
    1. 순서

    - 순서없음
    - 순서없음
        - 순서없음
        - 순서없음

    + 순서없음
    + 순서없음
        + 순서없음
        + 순서없음
```

1. 순서
1. 순서
   1. 순서
   1. 순서
1. 순서

- 순서없음
- 순서없음
  - 순서없음
  - 순서없음

* 순서없음
  - 순서없음
    - 순서없음
    - 순서없음

### 링크

글자를 누르면 해당 페이지로 이동하는 링크 입니다.
보통 현재 탭에서 열리는 것이 기본이지만, 새 탭에서 열릴 수도 있습니다.

```
[이름](링크)
[이름](링크 "설명")
[이름][참조]

[참조]: 링크
[참조]: 링크 "설명"
```

[Youtube](https://youtube.com/)

[Github](https://github.com/ "github)

[test] (../test)

[google][google Link]

[Naver][a]

[google Link]: google.com
[a]: https://naver.com '네이버로 이동'

### 이미지

이미지를 넣을 수 있습니다.
링크와 비슷하지만 앞에 !를 넣어야 합니다.

```
![대체텍스트](이미지 주소)
![대체텍스트](이미지 주소 "설명")
![대체텍스트](참조)
[참조]: 이미지 주소
[참조]: 이미지 주소 "설명"
```

![곰1](../assets/images.jpg)
![곰2](../assets/images2.jpg '곰돌이2')
![곰3][곰돌이3]
![곰4][곰돌이4]

[곰돌이3]: ../assets/images3.jpg
[곰돌이4]: ../assets/images4.jpg '곰돌이 4'

이미지에 링크 추가도 가능합니다.

```
[![github](../assets/images5.jpg)](https://github.com)
```

[![github](../assets/images5.jpg)](https://github.com)

### 표

표를 표현할 수 있습니다.
테이블 헤더를 구분하기위해 3개이상의 `-`하이픈을 을 사용하며 `:`을 추가해 셀(열)안에 내용을 정렬합니다.

가장 좌측과 우측의 `|`은 생략가능합니다 (플랫폼에 따라 안되는경우가 있으니 넣는걸 추천합니다.)

`|`은 테이블의 행을 의미합니다. | 문자로 사용하시려면 `\|`이스케이프를 사용하셔야합니다.

```
| 헤더 | 헤더 | 헤더 | 헤더 | 헤더 |
|---|---|---|---|---|
| 셀 | 셀 | 셀 | 셀 | 셀 |
| 셀 | 셀 | 셀 | 셀 | 셀 |

헤더 | 헤더 | 헤더| 헤더| 헤더
---|---|---|---|---
셀 | 셀 | 셀| 셀| 셀
셀 | 셀 | 셀| 셀| 셀
```

| 값         |                  의미                  |   기본값 |
| ---------- | :------------------------------------: | -------: |
| `static`   |     유형(기준) 없음 / 배치 불가능      | `static` |
| `relative` |       요소 자신을 기준으로 배치        |          |
| `absolute` | 위치 상 부모(조상)요소를 기준으로 배치 |          |
| `fixed`    |      브라우저 창을 기준으로 배치       |          |
| `sticky`   |       스크롤 영역 기준으로 배치        |          |
